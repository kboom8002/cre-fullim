# Mobile IM — "충분하지만 간편한" 투자설명서 컨셉 정립

---

## 1. 왜 Mobile IM인가 — 시장의 공백

```
DealCard                    ???                     Full IM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
카톡 복붙 5초           [여기가 비어있다]         18섹션 + 전문가 + Gate
블라인드 티저                                     Buyer-ready 승인
매수자 1차 관심 유도                              기관 투자자급 문서

생성 시간: 1분           생성 시간: ???           생성 시간: 수 시간~수 일
러닝 커브: 없음          러닝 커브: ???           러닝 커브: 높음
```

### 꼬마빌딩 시장의 현실

| 가격대 | 거래 비중 | IM 필요도 | 현재 대안 |
|---|---|---|---|
| 20~50억 | 매우 높음 | 낮음 — 딜카드면 충분 | ✅ DealCard |
| **50~150억** | **높음** | **중간 — 딜카드 이상, Full IM 미만** | **❌ 없음** |
| 150~500억 | 보통 | 높음 — Full IM 필수 | ✅ Full IM |
| 500억+ | 낮음 | 매우 높음 — 기관급 IM | ✅ Full IM + Expert |

> **50~150억 꼬마빌딩 매매에서 매수자가 "딜카드보다는 좀 더 자세히, 그런데 IM까지는 필요 없어"라고 하는 순간이 바로 Mobile IM의 시장.**

## 2. Mobile IM 컨셉 정의

### 한줄 정의

> **Mobile IM은 브로커가 현장에서 카톡으로 공유할 수 있는, 7섹션짜리 "충분한" 투자설명서다.**

### 핵심 원칙

| 원칙 | Full IM | Mobile IM |
|---|---|---|
| 섹션 수 | 18개 | **7개** (핵심만) |
| 생성 시간 | 수 시간 | **3분** |
| 입력 | SSoT Full (14레이어) | **SSoT Lite + 사진 5장** |
| 전문가 검토 | 필수 (섹션별 배정) | **선택** (원클릭 요청) |
| Gate Review | 8가지 Gate | **2가지** (Disclosure + Risk) |
| 출력 형식 | PDF/PPTX/Markdown | **모바일 웹 링크 + 카톡 공유** |
| 타겟 유저 | 기관·전문 매수자 | **개인·법인 소형 매수자** |
| 공유 방식 | 이메일 첨부 | **카톡 링크 한 줄** |

## 3. Mobile IM 7섹션 구조

Full IM의 18섹션에서 **꼬마빌딩 거래에서 매수자가 실제로 묻는 7가지**만 추출:

| # | 섹션 | 매수자 질문 | Full IM 대응 섹션 |
|---|---|---|---|
| 1 | **물건 개요** | "어떤 건물이야?" | `property_fact_sheet` |
| 2 | **입지·상권** | "위치가 어때?" | `location_access` |
| 3 | **임대 현황** | "임대 수익은?" | `rent_roll_lease_quality` |
| 4 | **수익 분석** | "수익률이 몇 %?" | `income_noi_yield_analysis` |
| 5 | **리스크 체크** | "뭘 확인해야 해?" | `risk_factors_dd_checklist` |
| 6 | **투자 포인트** | "왜 사야 해?" | `investment_thesis_buyer_fit` |
| 7 | **다음 단계** | "뭘 하면 돼?" | `appendix_next_steps` |

### Full IM 대비 제외 섹션과 이유

| 제외 섹션 | 이유 |
|---|---|
| `cover_confidentiality` | → 모바일 링크 자체에 간단 면책 포함 |
| `executive_summary` | → 물건 개요에 통합 |
| `building_condition_physical_review` | → 리스크 체크에 통합 |
| `debt_sensitivity_cash_flow` | → 50~150억 개인 매수자에게 과도 |
| `valuation_logic_comparables` | → 수익 분석에 간략 통합 |
| `legal_zoning_compliance` | → 리스크 체크에 통합 |
| `value_add_repositioning_scenario` | → 투자 포인트에 통합 |
| 나머지 8개 | → 꼬마빌딩 거래에서 초기 검토 시 불필요 |

## 4. Mobile IM UX — "카톡에서 시작, 카톡에서 끝"

### 생성 플로우 (3분)

```
[Step 1] DealCard 결과 화면에서
         "Mobile IM 만들기" 버튼 탭
              │
[Step 2] 부족한 정보 안내 (최소 3개)
         ✅ 가격대  ✅ 권역  ✅ 자산유형
         ⚠️ 임대수익 → "월세 총액 입력" 간단 폼
         ⚠️ 사진 → "대표 사진 3~5장 촬영" 카메라 연결
              │
[Step 3] "Mobile IM 생성" 탭 (AI 3분)
              │
[Step 4] 모바일 웹 미리보기
         → 7섹션 스와이프 카드
         → "카톡으로 보내기" 원클릭
```

### 공유 형태

```
카톡 메시지:
━━━━━━━━━━━━━━━━━━━━━
📋 성수권역 80억대 근생형 자산
   투자 메모 (Mobile IM)

✅ 수익률(추정): 약 4.8%
✅ 임대율: 85%
⚠️ 확인 필요: 임대차 만기, 주차

👉 상세 보기: https://js.im/m/abc123
━━━━━━━━━━━━━━━━━━━━━

[상세 보기 링크 클릭 시]
→ 모바일 최적화 7섹션 웹페이지
→ 스와이프로 섹션 이동
→ "상세 IM 요청" CTA (→ Full IM 전환)
```

## 7. 상품 체계 재정립 — 4-Tier → 5-Tier

```
Tier 0  DealCard (무료)
        ━━━━━━━━━━━━━━━
        카톡 복붙 → 블라인드 딜카드
        생성: 1분 | 공유: 카톡 | 매수자 1차 관심

Tier 1  Mobile IM (저가 또는 프리미엄 freemium) ← 🆕 NEW
        ━━━━━━━━━━━━━━━
        DealCard + 임대수익 + 사진 → 7섹션 모바일 IM
        생성: 3분 | 공유: 카톡 링크 | 매수자 2차 검토

Tier 2  AI + Expert Review (중가)
        ━━━━━━━━━━━━━━━
        Full IM 18섹션 + 전문가 1~3섹션 검토
        생성: 수 시간 | 공유: PDF | 매수자 실사 진입

Tier 3  Expert Full Build (고가)
        ━━━━━━━━━━━━━━━
        전체 전문가 검토 + Buyer-ready 승인
        생성: 수 일 | 공유: 패키지 | 기관 매수자

Tier 4  Deal Room Package (엔터프라이즈)
        ━━━━━━━━━━━━━━━
        IM + Q&A + Evidence Index + 통제된 공개
        생성: 프로젝트 | 공유: Deal Room | 다수 매수자 경합
```

### 전환 퍼널에서 Mobile IM의 역할

```
DealCard (100명 리드)
    ↓ 30% 전환
Mobile IM (30명 → "좀 더 자세히 볼까?")  ← 🆕 여기서 잡는다
    ↓ 20% 전환
Full IM 요청 (6명 → 진지한 매수자)
    ↓ 50% 전환
거래 성사 (3명)
```

> **Mobile IM이 없으면 DealCard → Full IM 사이에서 70%의 잠재 매수자가 이탈.** "딜카드는 봤는데, IM은 아직 없어요"라는 답변으로 관심이 식는 구간.

## 8. Mobile IM이 만드는 새로운 Unfair Advantage

### 8-1. "3분 IM" — 업계에 존재하지 않는 카테고리

전통 IM: 최소 2주, 비용 500만원+
→ Mobile IM: **3분, 브로커가 직접, 현장에서**

이 카테고리 자체가 존재하지 않으므로, **정의하는 자가 시장을 소유한다.**

### 8-2. 현장 즉시 생성 → 거래 속도 가속

```
[현장 방문 중]
브로커: (사진 5장 촬영, 임대 수익 간단 입력)
     → 3분 뒤 Mobile IM 생성
     → 매수자에게 카톡 즉시 공유
     → 매수자: "이 정도면 한번 봐볼게요"
     → 같은 날 현장 동행 일정 잡힘

[전통 방식]
브로커: "IM은 준비해서 보내드릴게요"
     → 2주 뒤 "아직 IM 안 됐어요?"
     → 매수자 관심 식음 → 딜 사망
```

### 8-3. Full IM 전환 파이프라인

Mobile IM 하단에:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 이 물건의 데이터 준비도: 52/100

상세 IM(18섹션)을 만들려면:
  □ 임대차 요약표 (→ 준비도 +20)
  □ 등기부등본    (→ 준비도 +15)
  □ 건물 사진 추가 (→ 준비도 +10)

[Full IM으로 업그레이드]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

> Mobile IM 자체가 **Full IM의 세일즈 도구**가 된다.
