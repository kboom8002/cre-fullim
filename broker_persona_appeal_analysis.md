# 60대 꼬마빌딩 브로커 관점 — 3시스템 소구 포인트 & 매력도 정밀 분석

## 1. 페르소나 정의: "JS부동산중개 김 실장"

| 항목 | 내용 |
|---|---|
| 나이 | 63세, 중개업 22년차 |
| 전문 분야 | 성수·강남 50~200억대 꼬마빌딩 매매 |
| 일일 업무 | 카톡 매물 정보 교환 → 현장 방문 → 오너 상담 → 매수자 연결 |
| 디지털 도구 | **카카오톡이 사실상 유일한 업무 도구** (네이버 부동산, 간헐적 엑셀) |
| 문서 작성 능력 | Word/PPT 기본 수준. 전문 IM은 외부 의뢰 또는 수기 작성 |
| 핵심 고민 | ① 매물 정보 정리에 시간이 너무 많이 걸림 ② 매수자에게 보낼 자료가 항상 부족 ③ 전문적 IM을 만들 역량/시간이 없음 ④ 젊은 브로커 대비 디지털 경쟁력 불안 |
| 거래 패턴 | 연 2~4건 성사, 건당 수수료 2,000~5,000만원 |
| 기술 수용도 | **낮음** — 새 앱 설치 저항, 복잡한 UI 포기, 카톡 내에서 해결 선호 |

## 2. 일상 업무 흐름과 시스템 매핑

```
06:00  카톡 단톡방에서 매물 정보 수신
       "성수동 000-00, 80억대 근생, 1층 카페 월세 800..."
       ──→ 🔵 DealCard: 카톡 메모 붙여넣기 → 1분 딜카드

09:00  오너와 전화 상담
       "우리 건물 팔려면 뭐가 필요해요?"
       ──→ 🔵 DealCard: Owner Readiness 체크리스트

10:00  매물 현장 방문, 사진 촬영
       핸드폰으로 30장 촬영
       ──→ 🟣 AiPage: 사진 업로드 → AI 분류 → 임대 페이지

13:00  매수자에게 카톡으로 매물 소개
       "김 대표님 조건에 맞는 물건이 나왔습니다"
       ──→ 🔵 DealCard: Buyer Memo → 카톡 복사

15:00  관심 매수자가 상세 자료 요청
       "IM 자료 보내주실 수 있어요?"
       ──→ 🟢 FullIM: 18섹션 자동 생성 + 전문가 검토

17:00  건물주에게 마케팅 현황 보고
       ──→ 🟣 AiPage: 건물주 보고서 (문의 현황 + 업종 관심도)
```

## 3. 시스템별 소구 포인트 분석

### 3-1. 🔵 DealCard — "카톡이 딜카드가 된다"

#### 즉각 와닿는 소구 포인트

| 순위 | 기능 | 브로커 언어로의 번역 | 매력도 |
|---|---|---|---|
| 🥇 | 카톡 메모 → 딜카드 | **"카톡에서 복사해서 붙여넣기만 하면 딜카드가 나온다"** | ★★★★★ |
| 🥈 | 블라인드 티저 자동 생성 | **"주소 안 알려줘도 되는 카드가 바로 만들어진다"** | ★★★★★ |
| 🥉 | 카톡 복사 버튼 | **"만들어진 문구를 그대로 카톡에 보낼 수 있다"** | ★★★★★ |
| 4 | 매수자 조건 정리 | **"김 대표님 원하는 조건을 카톡에서 복사해서 정리하면 자동으로 매칭 포인트가 나온다"** | ★★★★☆ |
| 5 | 오너 준비도 체크 | **"건물주한테 '이것만 준비하면 IM 만들 수 있다'고 구체적으로 말해줄 수 있다"** | ★★★☆☆ |

#### 채택 장벽 분석

| 장벽 | 심각도 | 근거 |
|---|---|---|
| 새 웹앱 접속 | 🟡 중간 | 카톡 외의 앱을 여는 것 자체가 허들. 단, 브라우저 링크라 설치 불필요 |
| 텍스트 붙여넣기 | 🟢 낮음 | 카톡에서 길게 눌러 복사 → 붙여넣기는 익숙한 동작 |
| 결과 이해 | 🟢 낮음 | "딜 포인트", "확인 필요" 등 브로커가 매일 쓰는 단어로 표현 |
| 정보 보안 우려 | 🟡 중간 | "내 매물 정보가 어디 가는 거야?" — 신뢰 구축 필요 |

#### 매력도 종합: ★★★★★ (5/5)

> **이 시스템이 가장 먼저, 가장 쉽게 채택될 것이다.** 이유: ① 입력이 "카톡 복사붙여넣기"로 제로 러닝커브, ② 결과가 "카톡으로 바로 보내는 문구"로 즉시 업무에 사용 가능, ③ 블라인드 티저는 60대 브로커가 평생 해온 "주소 안 알려주고 매물 소개"를 자동화한 것.

### 3-2. 🟣 Space AI Page — "사진이 마케팅이 된다"

#### 즉각 와닿는 소구 포인트

| 순위 | 기능 | 브로커 언어로의 번역 | 매력도 |
|---|---|---|---|
| 🥇 | 사진 벌크 업로드 → AI 분류 | **"사진 찍어서 올리면 알아서 정리해준다"** | ★★★★☆ |
| 🥈 | AI 임대 페이지 자동 생성 | **"내가 홈페이지를 만든 적이 없는데, 링크 하나가 나온다"** | ★★★★☆ |
| 🥉 | 카카오 홍보 카피 생성 | **"임차인한테 보낼 카톡 문구가 자동으로 나온다"** | ★★★★☆ |
| 4 | 문의 자동 캡처 | **"관심 있는 사람이 알아서 문의를 넣어둔다"** | ★★★☆☆ |
| 5 | 건물주 보고서 | **"건물주한테 '이만큼 마케팅했다'는 보고서를 보여줄 수 있다"** | ★★★★☆ |

#### 채택 장벽 분석

| 장벽 | 심각도 | 근거 |
|---|---|---|
| **임대 중심 시스템** | 🔴 높음 | 김 실장은 **매매** 전문. 임대 마케팅 도구의 즉각적 필요성 낮음 |
| 사진 업로드 과정 | 🟡 중간 | 30장 벌크 업로드는 익숙하지 않은 동작 |
| Visual Answer Album 개념 이해 | 🟡 중간 | "사진이 증거가 된다"는 새로운 개념 — 설명 필요 |
| Tenant Fit / Vibe Fit | 🔴 높음 | "분위기 적합성 분석"은 60대 브로커에게 과도하게 추상적 |

#### 매력도 종합: ★★★☆☆ (3/5)

> **매매 전문 브로커에게는 즉시 와닿지 않는다.** 단, 두 가지 시나리오에서 매력도가 급상승:
> - **시나리오 A**: 매각 전 임대 마케팅이 필요한 공실 건물 — "1층 공실 채워야 매각가가 올라가요"
> - **시나리오 B**: 건물주 관계 유지 도구 — "건물주한테 보고서 보여주면 전속 계약 유지에 도움"
> 
> **핵심 전략: 매매 브로커에게는 "임대 마케팅 도구"가 아니라 "매각가 올리기 도구"로 포지셔닝해야 함.**

### 3-3. 🟢 Full IM Studio — "전문가급 투자설명서가 자동으로"

#### 즉각 와닿는 소구 포인트

| 순위 | 기능 | 브로커 언어로의 번역 | 매력도 |
|---|---|---|---|
| 🥇 | 18섹션 IM 자동 생성 | **"IM을 외주 안 맡기고 내가 직접 만들 수 있다"** | ★★★★★ |
| 🥈 | Readiness Score | **"건물주한테 '지금 81점이고 이거 두 개만 더 주면 IM 나옵니다'라고 말할 수 있다"** | ★★★★☆ |
| 🥉 | 전문가 섹션별 검토 | **"세무사/감정평가사한테 전체 문서 안 보내고 필요한 부분만 부탁할 수 있다"** | ★★★★☆ |
| 4 | Buyer-ready Export | **"PDF로 떨어져서 매수자한테 바로 보낼 수 있다"** | ★★★★☆ |
| 5 | Gate Review 안전장치 | **"수익률 보장' 같은 위험한 말이 자동으로 걸러진다"** | ★★★☆☆ |

#### 채택 장벽 분석

| 장벽 | 심각도 | 근거 |
|---|---|---|
| **Full IM 필요성 인식** | 🟡 중간 | 50억 이하 물건은 IM 없이도 거래. 100억+ 물건에서 IM의 가치 극대화 |
| 시스템 복잡도 | 🔴 높음 | 18섹션, Gate Review, Expert Patch — 기능이 많고 용어가 전문적 |
| 데이터 입력 부담 | 🟡 중간 | Readiness Score를 올리려면 자료를 모아야 함 (등기부, 임대차 등) |
| "AI가 쓴 IM을 믿을 수 있나?" | 🟡 중간 | 전문가 검토 프로세스가 있다는 점이 해소 포인트 |
| 비용 대비 가치 인식 | 🟢 낮음 | 기존 IM 외주 비용 (500~2,000만원) 대비 압도적 가성비 |

#### 매력도 종합: ★★★★☆ (4/5)

> **100억+ 물건에서는 "게임 체인저"이지만, 일상적으로 매일 쓰는 도구는 아니다.** 연 2~4건의 큰 딜에서 IM이 필요할 때 "이런 게 있었지" 하고 꺼내 쓰는 도구. 가치 인식은 매우 높지만, **사용 빈도가 낮다는 점이 채택의 최대 리스크.**

## 5. "이 브로커가 실제로 쓸까?" — 채택 시나리오

### Phase 1: 즉시 채택 — DealCard (Day 1)

```
김 실장이 아침에 카톡 단톡방에서 매물 정보를 받는다.
"성수동 000-00, 80억대 근생..."

예전: 머릿속으로 정리 → 카톡으로 매수자에게 구두 설명
지금: 복사 → DealCard 붙여넣기 → "딜카드 만들기" → 카톡 복사

김 실장: "이거 편하네. 카톡에서 복사해서 넣으면 깔끔한 카드가 나오잖아."
```

> **성공 조건**: 5초 내 결과, 카톡 복사 한 번에 완료, 앱 설치 불필요

### Phase 2: 점진 채택 — Full IM (Month 1~2)

```
매수자: "김 실장님, 이 건물 IM 자료 있어요?"
예전: "IM은... 제가 따로 준비해서 보내드리겠습니다" (실제로는 못 만듦)
지금: DealCard에서 "Full IM 가능 여부 확인" 클릭 → 81점
      → "두 가지만 더 준비하면 됩니다" → 오너에게 자료 요청
      → Full IM Studio에서 18섹션 자동 생성

김 실장: "이런 걸 만들어주는 데가 있었어? 예전에는 500만원 줘야 했는데."
```

> **성공 조건**: DealCard에서 자연스럽게 연결, Readiness Score가 "무엇이 부족한지" 명확히 안내

### Phase 3: 선택 채택 — Space AI Page (Month 3+)

```
오너: "1층 공실이 안 나가요. 임대료 낮춰야 하나?"
김 실장: "제가 임대 마케팅 페이지를 만들어볼게요."
      → 현장 사진 30장 촬영 → AiPage 업로드
      → AI 임대 페이지 생성 → 카톡으로 임차인에게 공유
      → 문의 3건 접수 → 건물주 보고서 전달

김 실장: "건물주한테 보고서 보여주니까 '역시 김 실장이 다르다'고 하더라."
```

> **성공 조건**: 매매가 아닌 "건물주 관계 관리" 도구로 포지셔닝

## 6. 최대 리스크와 대응 전략

### 리스크 1: "나는 컴퓨터를 잘 못해요"

| 대응 | 구현 상태 |
|---|---|
| 모바일 퍼스트 UI (엄지 조작) | ✅ 3개 시스템 모두 |
| 큰 글씨 (본문 15~16px) | ✅ DealCard |
| "카톡 복사" 원클릭 | ✅ DealCard, AiPage |
| 복잡한 용어 대신 일상어 | ✅ DealCard ("딜 포인트 보기", "확인 필요") |
| FullIM 용어 간소화 | ⚠️ 개선 필요 ("Gate Review" → "검토 통과 여부") |

### 리스크 2: "내 매물 정보가 유출되면?"

| 대응 | 구현 상태 |
|---|---|
| 자동 마스킹 안내 카드 | ✅ DealCard (숨긴 정보 목록 표시) |
| 3중 Disclosure Guard | ✅ 3개 시스템 |
| P0 자동 차단 | ✅ FullIM |
| 신뢰 문구 | ✅ "정확한 주소, 임차인명은 자동으로 숨겨집니다" |

### 리스크 3: "AI가 쓴 걸 믿을 수 있어?"

| 대응 | 구현 상태 |
|---|---|
| 확인됨/가설/확인필요 3단계 표시 | ✅ DealCard |
| 면책 문구 자동 삽입 | ✅ 3개 시스템 |
| 전문가 검토 프로세스 | ✅ FullIM |
| Financial Guardrail | ✅ FullIM ("수익률 보장" 자동 차단) |
